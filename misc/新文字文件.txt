文件一：需求規格書
PatternWeaver AI 預測模組 - 需求規格
1. 專案背景
在現有的 PatternWeaver 交易圖表系統基礎上，新增 AI 預測功能模組。
2. 核心需求
2.1 功能需求

分析開盤前5分鐘（21252225）的技術指標數值 然後以開盤之後的兩個小時的價格走向作為評分標準
使用 AI 模型預測最佳交易策略
預測內容包含：

交易方向（LongShort）
進場時機（Entry）
出場時機（Exit）
預期波動率



2.2 技術指標需求

實作 25+ 種技術指標（參考 Jesse Framework）
每個指標保留最新 50 個數值
不需要即時更新（批次計算即可）
不需要畫線顯示（只需要數值）

2.3 資料需求

使用現有的 MNQ 歷史資料（2019-2024）
生成每個交易日的指標快照
記錄開盤後的實際走勢作為訓練標籤

3. 系統整合需求
3.1 與現有系統的關係

作為獨立模組運作
共用現有的資料載入機制
可選擇性地在介面顯示預測結果

3.2 資料儲存

指標快照儲存為 JSON 格式
訓練資料獨立管理
模型檔案版本控制

4. 非功能需求
4.1 效能需求

批次處理所有歷史資料

4.2 準確度期望

初期目標：優於隨機（50%）
可接受範圍：55-65% 準確率
持續優化目標

5. 開發限制

使用 Python 生態系統
保持程式碼模組化

6. 交付項目

指標計算模組
快照生成器
標籤生成器
AI 訓練腳本
預測 API
效能評估報告
